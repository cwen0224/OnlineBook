# 系統 Bug 與已知問題追蹤

## 1. 初始黑畫面/左側淨空 Bug (Empty Left-Viewport Void)

**現象描述**：
當剛進入繪本（即停留在第一頁），或是將書往前翻回第一頁時，畫面的左半部會變成一片深灰色（或黑色）的實色區域，看起來像是一半的畫面有 bug 出不來。

**發生原因**：
目前的 3D 立體繪本引擎是以「對開式 (Spread)」為核心架構。書本的轉軸（書脊）被嚴格鎖定在螢幕的絕對正中央 (X=50%)。
所有的紙張在一開始都是以 `width: 50%` 掛在螢幕的右半邊。這代表在讀者「將第一張紙往左翻」之前，整個螢幕的左半邊是完全沒有任何紙張覆蓋的。因此，使用者的視線會直接穿透右側的書本左緣，看到網頁最底層桌面的 `background-color` (目前預設為 #222 )。讀者會誤以為這是畫面破圖死角，但在 3D 模型邏輯裡，這其實只是因為桌子的左邊還沒有放置任何一頁的書。當往左翻動後，翻過去的紙張就會覆蓋左半部，問題隨之消失。

**出現機率**：
100%。只要返回第一張葉片（無葉片位於左側時），必定出現。

**建議升級與修復方案（擇一即可）**：
1. **加入全版桌面情境底圖**：讓 `body` 掛上一張具有質感的木桌材質 `background-image`。只要不是純黑，讀者就會意識到那是「後面的桌子」而非「破掉的畫面」。
2. **實作實體書裝訂封底 (Hardcover Back)**：透過 DOM 結構增設一層永遠躺在左邊、不參與陣列翻動的「封底底板」，讓左半邊永遠有一層卡紙墊底。
3. **攝影機動態平移視角 (Camera Paning)**：當書本處於閉合狀態時，將整個 `#book` 容器透過 CSS 平移，讓第一頁能獨自置中於全螢幕；當使用者開始翻頁時，再透過動畫平滑地將書本連同視角滑回對開的中央狀態。
